# 🧪 Web Automation with Selenium & CI Pipeline Integration

This project demonstrates a web automation testing framework built with Selenium WebDriver, integrated with a Continuous Integration (CI) pipeline to automatically execute tests on every code change.

It is designed to showcase:

- UI automation using Selenium<br>
- Scalable test structure<br>
- CI/CD integration for automated execution<br>
- Real-world DevOps workflow for QA automation

<br><br>
# 🚀 Project Overview

The goal of this project is to automate browser-based test scenarios and ensure continuous validation through a CI pipeline.

Every push or pull request triggers:

- Automated Selenium test execution
- Test reporting in pipeline logs
- Build status feedback (pass/fail)

This ensures early detection of UI issues and improves software reliability.
<br><br>
# 🏗️ Tech Stack
- Python
- Selenium WebDriver
- Test framework (PyTest)
- Git & GitHub
- CI/CD: Jenkins
<br><br>
# ⚙️ How It Works
- Test cases are written using Selenium WebDriver
- Tests follow Page Object Model (POM) structure
